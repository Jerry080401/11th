自由軟體的四大定義

The Free Software Definition

[1] 根據 Richard Stallman 和自由軟體基金會（FSF）的定義， 自由軟體賦予使用者四種自由特質：

  1. 不論目的為何，可以自由使用該軟體
  2. 可以自由再次分享（散布）該軟體以幫助其他人
  3. 可以自由研究軟體的運作，並可以修改它來貼近自己的需求
  4. 可以自由地該良軟體，修改後的版本也能自由散步分享、惠及公眾

同時具有以上四種自由特質的軟體，就被稱作「自由軟體」。

[2] 自由軟體的使用者，可以不限目的／時間／地點／對象，不以獲利為主要目標，自由散布軟體的原版或改良版。

如果使用者不必問任何人或是支付任何的授權費用從事這些行為，就表示自由軟體所賦予的自由權利。

開放原始碼

Open Source

[4] 在 Open Source Initiavite 網站中有對開放原始碼做出定義， 這邊取3個比較重要的：

  1. 自由散布：不用經過他人的許可即可以自由地散布軟體
  2. 開放源碼：散布的軟體會附上完整的原始碼，或必須是有人們容易取得原始碼的管道
  3. 衍生著作：人們可以自由地修改原始碼，並依同一授權條款再發布

[5] 大部分的人會將開放原始碼與自由軟體搞混，因為兩者重疊的地方很多。 雖然絕大部分的開源軟體也都符合自由軟體的定義， 但實際上兩者個規範不同，只要符合各自的規範，就屬於各自規範底下的軟體。 而自由軟體的規範比開放原始碼的定義更為嚴謹，因此所有的自由軟體也都是開放原始碼。

開放授權&公眾授權

Open License & Public License

「開放授權」讓著作權所有者只保留部分權力，允許其作品可以被自由且不羨慕地的使用、修改和散布。 任何額外附加的條寬都不可牴觸授權原則。與「公共授權」相比，「開放授權」自由度更大。
這些軟體程式碼常用的開放授權條款，從最嚴格的 GPL 類，到最寬鬆的 MIT／BSD 類， 只要和上述的定義不相悖，都可以被視為開放授權。

從最嚴格到最寬鬆：
  • AGPL
  • GPL
  • LGPL
  • EPL／CPL
  • MPL／CDDL
  • Apache Lincese
  • MIT／BSD

「公眾授權」是創作者自願鬆綁著作權法嚴謹的限制，讓作品的應用更自由， 其再利用的自由程度取決於個別條款內容，人然可限制能不能夠商用或修改。

[6] 著名的CC授權：
  • CC-0
  • CC-BY
  • CC-BY-SA
  • CC-BY-NC
  • CC-BY-NC-SA
  • CC-BY-ND
  • CC-BY-NC-ND
各個代表的意思：
  • BY 姓名標示
  • SA 相同方式分享
  • NC 非商業性使用
  • ND 禁止改作
